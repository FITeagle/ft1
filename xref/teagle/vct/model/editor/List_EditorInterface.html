11   *

16   * not possible by defining a class that extends (subclasses) these two. This is
17   * solvable either by having the two configurations as fields in a new class
18   * (containment) or by defining VirtualNode and Router as interfaces. Still, in
19   * many cases abstract classes and subclassing are enough.
20   * 
21   * The reason this template parameter needs to be passed is because inferring
22   * the parameter of an ArrayList<Something> through reflection is to my
23   * knowledge not possible.
